Agent Skills bring the full power of Unrag's documentation, patterns, and best practices directly into your AI coding assistant. Instead of context-switching between docs and your editor, your AI assistant becomes an Unrag expert that understands the entire API, all 12 embedding providers, extractors, connectors, and production patterns.

## What are Agent Skills?

Agent Skills are structured knowledge packages designed for AI coding assistants like Claude Code, Cursor, Windsurf, and others. They provide:

- **Complete API knowledge** — Every type, method, and configuration option
- **All 12 embedding providers** — OpenAI, Google, Cohere, Voyage, Ollama, and more
- **All 12 extractors** — PDF, image, audio, video, and file processing
- **4 connectors** — Notion, Google Drive, OneDrive, Dropbox
- **Production patterns** — Search endpoints, multi-tenant scoping, chat integration
- **Troubleshooting guides** — Common issues and debugging strategies

When you install the Unrag skill, your AI assistant can:

- Write correct Unrag code without hallucinating APIs
- Configure embedding providers with the right environment variables
- Set up store adapters (Drizzle, Prisma, Raw SQL) properly
- Implement asset extraction pipelines
- Build production-ready search endpoints
- Debug issues with accurate troubleshooting steps

## Installation

Install the Unrag agent skill using the `add-skill` CLI by Vercel Labs:

```bash
npx add-skill betterstacks/unrag
```

This installs the skill into your project's `.claude/skills/` directory (or equivalent for your AI assistant).

<Callout type="info">
  The skill is designed for Claude Code but works with any AI assistant that supports the skills format.
</Callout>

## How It Works

When you ask your AI assistant to help with Unrag, the skill provides contextual knowledge:

```
You: "Help me set up OpenAI embeddings with Drizzle"

AI: I'll help you configure OpenAI embeddings with the Drizzle store adapter.

First, install the dependencies...
[Provides accurate, working code]
```

The AI can reference:

1. **Exact type signatures** — No guessing at API shapes
2. **Environment variables** — Knows `OPENAI_API_KEY` not `OPENAI_KEY`
3. **Configuration patterns** — Proper `defineUnragConfig()` structure
4. **Best practices** — Connection pooling, error handling, performance tuning

## Skill Structure

The skill is organized into focused reference files:

```
skills/unrag/
├── SKILL.md                    # Main skill overview
└── references/
    ├── api-reference.md        # Complete type definitions
    ├── embedding-providers.md  # All 12 providers
    ├── store-adapters.md       # Drizzle, Prisma, Raw SQL
    ├── extractors.md           # All 12 extractors
    ├── connectors.md           # Notion, Drive, OneDrive, Dropbox
    ├── batteries.md            # Reranker, Eval, Debug
    ├── cli-commands.md         # init, add, upgrade, doctor, debug
    ├── patterns.md             # Common recipes
    └── troubleshooting.md      # Debug and issues
```

This structure ensures the AI can quickly find relevant information without overwhelming context.

## Keeping Skills Updated

The skill version tracks Unrag releases:

- **Skill Version:** 1.0.0
- **Unrag CLI Version:** 0.3.2
- **Config Version:** 2

Update your skill when Unrag releases new features:

```bash
npx add-skill betterstacks/unrag --update
```

Agent Skills represent a new way to build software—where AI assistants have deep, accurate knowledge of the tools you use. Instead of copying documentation snippets or hoping the AI remembers the right API, skills provide structured, version-controlled knowledge.

2. **Write correct code** — AI knows the actual types and patterns
3. **Debug effectively** — Accurate troubleshooting from the skill
